# Codex Quota HUD 项目开发说明

本文面向未来维护者和 AI 编程代理，用于说明当前项目结构、关键行为、生命周期管理、测试要求和安全边界。由于项目已经从单一 Codex HUD 扩展为 Codex + 可选 Managed AGY provider，后续修改必须保持模块化，避免把所有逻辑重新塞回 `MainHudForm.cs`。

---

## 1. 项目概述

Codex Quota HUD 是一个 Windows 原生桌面 HUD 工具，使用 **C# + WinForms** 开发。

当前核心能力：

```text
读取本机额度 -> 统一成 provider/bucket 模型 -> 以 HUD 展示
```

当前 provider：

- `Codex`：通过本机 Codex CLI `app-server --listen stdio://` 读取 `7d` / `5h` 额度。
- `AGY · Gemini`：可选启用，通过 HUD 托管启动的 Antigravity CLI `agy.exe` 读取 Gemini `7d` / `5h` 额度。

产品仍然是小型 HUD，不是完整账号仪表盘。

## 5. 核心架构

### 5.1 Provider 模型

不要让 UI 直接依赖 Codex 或 AGY 的原始协议字段。数据流应该是：

```text
IQuotaProvider
  -> ProviderQuotaSnapshot
    -> QuotaBucketSnapshot[]
      -> UI 控件展示
```

`ProviderQuotaSnapshot` 应包含：

- `ProviderId`
- `DisplayName`
- `Subtitle`
- `Source`
- `Status`
- `Buckets`
- `UpdatedAt`
- `CheckedAt`
- `ChangedAt`
- `ErrorMessage`
- `IsManagedProcess`
- `ProcessId`
- `Port`

`QuotaBucketSnapshot` 应包含：

- `Id`
- `Label`
- `ShortLabel`
- `RemainingPercent`
- `ResetAt`
- `RawResetTimeUtc`

UI 层统一显示 `7d` / `5h`，即使 AGY 内部实际字段是 `gemini-weekly` / `gemini-5h`。

## 6. 主要模块说明

### 6.1 `Program.cs`

职责：

- 初始化 WinForms。
- 创建并运行 `MainHudForm`。

不要在此处加入额度读取、provider 生命周期、UI 细节或发布逻辑。

---

### 6.2 `MainHudForm.cs`

主 HUD 窗口。

职责：

- 管理展开态和吸附/折叠态。
- 管理拖动、靠顶吸附、悬停展开。
- 管理托盘图标和托盘菜单。
- 调用 `QuotaPoller` 刷新 provider snapshot。
- 复用 `ProviderCardControl` 和 `CollapsedProviderRowControl`。
- 显示 AGY 错误 toast。
- 管理退出流程。

重要规则：

- 不要每次刷新都 `Controls.Clear()` 再重建所有控件。
- Provider card / collapsed row 应尽量复用。
- 只有 provider 数量变化、展开/折叠状态变化、footer 高度变化时才重新 layout。
- 普通额度刷新只更新文字、时间、进度条 percent。
- 退出时必须先停止 timer、取消刷新、释放 provider，再关闭托盘图标和窗体。

---

### 6.3 `QuotaPoller.cs`

统一调度 provider 刷新。

职责：

- 永远刷新 Codex。
- 只有设置启用时才刷新 AGY。
- 使用 refresh gate 避免并发刷新。
- 应支持退出时 cancellation。
- Dispose 时释放 provider。

建议：

- `RefreshAsync` 接受 `CancellationToken`。
- 退出时先 cancel，再 dispose。
- `DisposeAsync` 内部 await 应使用 `ConfigureAwait(false)`，避免 WinForms UI 线程死锁。

---

### 6.4 `CodexQuotaProvider.cs`

Codex provider 适配层。

职责：

- 调用 `CodexQuotaReader.ReadAsync()`。
- 将 Codex 专用 `QuotaSnapshot` 映射为统一 `ProviderQuotaSnapshot`。
- 将 `SevenDay` / `FiveHour` 映射为 `7d` / `5h` bucket。
- 维护 `ChangedAt`。

---

### 6.5 `CodexQuotaReader.cs`

读取 Codex CLI 额度。

当前模式是 one-shot 子进程：

```text
每次刷新
  -> 启动 codex app-server --listen stdio://
  -> 发送 initialize
  -> 发送 account/rateLimits/read
  -> 读取 JSON response
  -> 解析 result
  -> kill 子进程
```

路径解析顺序：

1. `CODEX_CLI_PATH`
2. `%LOCALAPPDATA%\OpenAI\Codex\bin\codex.exe`
3. `%LOCALAPPDATA%\OpenAI\Codex\bin\` 下的版本化子目录
4. PATH 中的 `codex`

注意：

- 该 reader 可以使用隐藏子进程。
- 失败时返回短错误，不要把大段 stderr 塞进 HUD。
- 日志不要记录敏感信息。

---

### 6.6 `ManagedAgyQuotaProvider.cs`

AGY provider 适配层。

职责：

- 检查 AGY 是否启用。
- 调用 `ManagedAgyProcess.EnsureRunningAsync()` 确保 HUD 托管的 AGY 正在运行。
- 发现或复用本地 quota endpoint 端口。
- 调用 `RetrieveUserQuotaSummary`。
- 解析 `gemini-weekly` / `gemini-5h`。
- 映射为 UI 的 `7d` / `5h`。
- 维护 `CheckedAt` / `ChangedAt`。
- 在 endpoint 不可用时返回 Offline / Failed snapshot。

推荐优化：

- 复用 provider 级别 `HttpClient`，避免每次刷新都创建。
- endpoint 不可用时设置 backoff，避免每轮刷新都完整 discovery 15 秒。
- 连续 not ready 后可关闭托管 AGY，并提示用户手动运行一次 `agy`。

---

### 6.7 `ManagedAgyProcess.cs`

管理 HUD 自己启动的 `agy.exe`。

职责：

- 解析 `agy.exe` 路径。
- 使用专用工作目录启动 AGY。
- 支持隐藏启动。
- drain stdout/stderr，避免管道阻塞。
- 记录托管进程的 PID、启动时间、路径、工作目录。
- HUD 退出时只关闭自己启动的 AGY。
- AGY 意外退出后，下次刷新可以重新启动。

重要规则：

```text
禁止 kill all agy.exe
只允许关闭 HUD 自己启动且身份匹配的 agy.exe
```

判断托管进程时建议检查：

- PID 是否是记录的 PID。
- 进程名是否为 `agy`。
- StartTime 是否匹配。
- MainModule 路径是否匹配。
- 工作目录或启动标记是否匹配。

如果保留 `CloseManagedAgyOnExit` 设置，退出时必须尊重它；如果不打算开放该设置，应删除未使用字段，避免误导维护者。

---

### 6.8 `AgyEndpointDiscovery.cs`

发现 AGY 本地监听端口。

职责：

- 根据 PID 查找 TCP Listen 端口。
- 对候选端口进行 quota endpoint probe。

当前可用方式是调用 `netstat -ano -p tcp` 并解析 PID 对应端口。

注意：

- 不要假设固定端口。
- 如果 endpoint 连续不可用，应 backoff，避免频繁启动 `netstat`。
- discovery 必须支持 cancellation。

---

### 6.9 `AntigravityQuotaParser.cs`

解析 AGY quota JSON。

核心字段：

```text
response.groups[].buckets[]
bucketId == gemini-weekly
bucketId == gemini-5h
remainingFraction
resetTime
```

映射规则：

```text
gemini-weekly -> UI 7d
gemini-5h     -> UI 5h
remainingPercent = remainingFraction * 100
resetTime UTC -> 本地时间显示
```

解析器必须宽松：

- 不要强依赖 `description`。
- 不要强依赖 `displayName`。
- 不要强依赖 `window`。
- 兼容 `remainingFraction` 在 bucket 顶层或 `remaining.remainingFraction` 下。

---

### 6.10 `ProviderCardControl.cs`

展开态 provider 卡片。

职责：

- 显示 provider 标题，例如 `Codex`、`AGY · Gemini`。
- 显示右上角状态，例如 `Updated HH:mm` / `Checked HH:mm` / `Offline`。
- 显示 `7d` / `5h` 进度条。
- 显示 reset 时间。
- 仅在失败状态显示 footer 错误信息。

规则：

- 正常状态不显示 `Source Managed AGY` footer。
- 错误 footer 只显示短错误。
- 更新 snapshot 时只更新变化的 label / bar。
- 如果 footer 显示状态变化导致高度变化，通知主窗体重新 layout。

---

### 6.11 `CollapsedProviderRowControl.cs`

折叠态 provider 行。

推荐布局：

```text
Codex  [7d] [5h]
AGY    [7d] [5h]
```

规则：

- 正常状态显示两个紧凑进度条。
- Offline / Failed 状态显示短状态，例如 `AGY offline`。
- 不显示百分比。
- 不显示 reset 时间。

---

### 6.12 `QuotaBarControl.cs`

自绘进度条。

职责：

- 绘制 track。
- 绘制 fill。
- 绘制边框。
- 绘制 label 或百分比。
- 根据 fill 色自动选择文本颜色，避免浅色背景下白字看不清。

规则：

- 低百分比不要强行最小宽度等于高度，否则 5% 和 11% 会看起来一样。
- setter 中应比较旧值和新值，没变化不 `Invalidate()`。
- 默认 `5h` 色为 `#C2410C`，不是浅黄色。

---

### 6.13 `SettingsForm.cs`

设置窗口。

职责：

- 自动刷新间隔设置。
- Enable Antigravity 开关。
- hex 颜色输入。
- 点击颜色块打开系统调色盘。
- Reset / Cancel / Save。

规则：

- 不再使用单独的 `Change` 按钮。
- 色块本身就是调色盘入口。
- 保存时验证 `#RRGGBB`。
- Settings 窗口应保持紧凑。

## 8. 刷新行为

自动刷新选项：

```text
30 sec
1 min
5 min
10 min
20 min
```

默认：

```text
1 min
```

刷新规则：

- 手动刷新通过右键菜单 `Refresh Now`。
- 自动刷新由 WinForms Timer 触发。
- 不允许并发刷新。
- 刷新时只更新变化的控件。
- 退出时必须取消正在进行的刷新。

防闪烁要求：

- Provider card / row 只创建一次并复用。
- `QuotaBarControl.Percent` 相同则不重绘。
- Label 文本相同则不重新赋值。
- 不要每次刷新都改变 Form Size。
- 只有 layout 真实变化时才 `ShowCurrentView(force: true)`。

## 9. 退出与生命周期要求

这是当前项目最重要的稳定性规则。

### 9.1 不要在 Dispose 中同步等待异步释放

禁止在 UI 线程 `Dispose()` 中使用：

```csharp
_poller.DisposeAsync().AsTask().GetAwaiter().GetResult();
```

这可能导致：

- 托盘图标已经消失。
- HUD 残留在桌面。
- 窗口无法移动/选中。
- 进程未退出。
- exe 被锁定，导致下一次 `dotnet run` 无法覆盖。

### 9.2 推荐退出流程

```text
ExitApplicationAsync
  -> 设置 _isExiting = true
  -> 禁用托盘菜单，避免重复点击
  -> 停止 refresh timer
  -> Cancel 应用级 CancellationTokenSource
  -> await 当前 refresh / provider dispose
  -> 关闭 HUD 自己启动的 AGY
  -> 隐藏并释放 NotifyIcon
  -> Close / Application.ExitThread
```

所有后台 async dispose 链路建议使用：

```csharp
await SomethingAsync().ConfigureAwait(false);
```

避免捕获 WinForms UI 同步上下文。

### 9.3 AGY 退出要求

- HUD 退出时只关闭自己启动的 AGY。
- 不允许 kill 用户手动打开的 AGY。
- 如果 AGY 意外退出，下一轮刷新应重新启动。
- 如果 AGY endpoint 长期不可用，应 backoff，并提示用户手动运行一次 `agy`。

## 11. 日志与安全

允许写入简短 debug 日志，但必须避免敏感信息。

不要记录：

- 完整 CSRF token。
- 完整 command line。
- 完整 raw JSON。
- 账号邮箱。
- 用户 prompt。
- 可能包含凭据的环境变量。

AGY CLI provider 当前不需要 CSRF header，但未来如果新增 App Local provider，要特别注意 token 脱敏。

TLS 证书绕过只允许用于：

```text
https://127.0.0.1:<port>
```

不要全局关闭 TLS 校验。

- Enable Antigravity 默认关闭。

### 13.4 Exit 测试

这是必须测试项。

- 右键托盘 `Exit` 后：
  - 托盘图标消失。
  - HUD 窗口消失。
  - `CodexQuotaHud.exe` 进程退出。
  - HUD 托管的 `agy.exe` 退出。
  - 不留下无法移动/无法选中的残影窗口。
- 退出后立即执行 `dotnet run` 不应出现 exe 被锁定。

如果出现：

```text
CodexQuotaHud.exe is being used by another process
```

说明旧进程未退出，应检查退出流程。

## 16. 安全修改策略

结构性调整：

1. 先确认当前 build 通过。
2. 只移动文件，不改逻辑。
3. 再 build。
