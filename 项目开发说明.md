# Codex Quota HUD 项目开发说明

本文档面向未来维护者和 AI 编程代理，说明本项目当前的产品方向、模块边界、UI 行为、Codex / AGY 额度读取方式，以及继续开发时应遵守的安全策略。

---

## 1. 项目概述

Codex Quota HUD 是一个轻量级 Windows 桌面 HUD，使用 **C# + WinForms** 构建。

它的目标是用一个小型、原生、置顶的 HUD 展示本机额度信息：

- Codex `7d` 剩余额度。
- Codex `5h` 剩余额度。
- 可选 AGY / Antigravity Gemini `7d` 剩余额度。
- 可选 AGY / Antigravity Gemini `5h` 剩余额度。
- reset 时间。
- 最近检查 / 更新时间。

核心产品思路：

```text
读取本地额度 -> 统一成 provider snapshot -> 用原生 WinForms HUD 展示。
```

当前产品方向不是“大型额度分析工具”，而是：

```text
一个小巧的 Windows 原生 Codex / AGY 额度 HUD。
```

---

## 2. 非目标

除非用户明确要求，否则不要添加：

- 额度使用节奏建议。
- 长期历史分析。
- quota 预测。
- 复杂图表。
- 账号管理。
- 云同步。
- 自动更新系统。
- 多平台 UI 框架迁移。
- Electron / Tauri / Wails / WebView 重写。
- 大型 dashboard。

项目应保持简单、稳定、便于 AI 代理维护。

## 3. 技术栈

- 语言：C#
- UI：WinForms
- 目标框架：`net8.0-windows`
- 运行目标：Windows x64
- 构建：`dotnet build`
- 发布：`dotnet publish`
- CI：GitHub Actions on `windows-latest`

选择 WinForms 的原因：

- 原生。
- 依赖少。
- 适合托盘、置顶窗口、自定义绘制、窗口吸附。
- 对 AI 编程代理来说易于阅读和修改。

## 5. 核心数据模型

### 5.1 Provider snapshot

UI 不应直接依赖 Codex 或 AGY 的原始协议字段，而应使用统一模型。

典型概念：

```text
QuotaProviderSnapshot
QuotaBucket
QuotaStatus
```

Provider snapshot 建议包含：

```text
ProviderId          codex / agy
DisplayName         Codex / AGY
Subtitle            可选，例如 Gemini
Status              Disabled / Refreshing / Ok / Failed / Offline
Buckets             额度桶列表
UpdatedAt           UI 更新时间
CheckedAt           最近一次成功检查时间
ChangedAt           额度数值最近变化时间
ErrorMessage        简短错误信息
IsManagedProcess    是否由 HUD 托管进程提供
```

### 5.2 Quota bucket

Quota bucket 建议包含：

```text
Id                  codex-7d / codex-5h / gemini-weekly / gemini-5h
Label               7d / 5h
ShortLabel          7d / 5h
RemainingPercent    剩余百分比
ResetAt             reset 时间
ColorRole           SevenDay / FiveHour
```

注意：

- UI 中 Codex 与 AGY 都显示为 `7d` 和 `5h`。
- AGY 的 `gemini-weekly` 映射为 UI 的 `7d`。
- AGY 的 `gemini-5h` 映射为 UI 的 `5h`。

## 6. 主要模块说明

### 6.1 `Program.cs`

职责：

- 初始化 WinForms。
- 启动主 HUD。
- 不放置额度读取、复杂 UI 布局或发布逻辑。

### 6.2 `MainHudForm.cs`

职责：

- 管理主窗口。
- 管理展开态 / 折叠态。
- 管理顶部吸附。
- 管理悬停展开。
- 管理托盘图标和右键菜单。
- 调用刷新逻辑。
- 根据 provider snapshot 更新 UI。

重要规则：

- 不要在每次刷新时重建整个界面。
- provider card 应尽量创建一次，后续只更新变化字段。
- provider 数量变化时才重新 layout。
- 折叠 / 展开状态变化时才调整窗口大小。
- 普通额度刷新不应导致整个 HUD 闪烁。

### 6.3 `ProviderCardControl.cs`

展开态 provider 模块控件。

职责：

- 显示 provider 标题，例如 `Codex`、`AGY · Gemini`。
- 显示状态时间，例如 `Updated 20:52` 或 `Checked 20:52`。
- 显示 `7d` / `5h` quota rows。
- 显示 reset 标记 `R` 和 reset 时间。
- 显示简短错误状态。

规则：

- 正常状态下不要显示 `Source Managed AGY`。
- 调试来源信息可放到 tooltip 或 debug log。
- provider 模块之间应有边界感，例如圆角边框。

### 6.4 `CollapsedProviderRowControl.cs`

折叠态 provider 行控件。

职责：

- 显示 provider 短名，例如 `Codex`、`AGY`。
- 显示两个紧凑额度条：`7d` 和 `5h`。
- 不显示百分比。
- 不显示 reset 时间。

示例：

```text
Codex   [7d] [5h]
AGY     [7d] [5h]
```

### 6.5 `QuotaBarControl.cs`

自绘进度条控件。

职责：

- 绘制轨道背景。
- 绘制轨道边框。
- 绘制填充区域。
- 绘制 label 或百分比文字。

推荐默认颜色：

```text
7d color:          #4EA1FF
5h color:          #C2410C
Track color:       #303740
Track border:      #7A8796
Background color:  #181C22
Text color:        #F2F4F8
```

行为规则：

- 低百分比时不要强制填充宽度等于 bar 高度，避免 `5%` 看起来像 `11%`。
- `Percent` setter 里应先比较旧值和新值；未变化时不要 `Invalidate()`。
- 根据填充色自动选择文字颜色：浅色填充用深色文字，深色填充用浅色文字。
- 控件启用双缓冲，减少闪烁。

### 6.6 `SettingsForm.cs`

设置窗口。

职责：

- 显示自动刷新间隔。
- 显示是否启用 Antigravity。
- 显示颜色设置。
- 支持 Reset / Save / Cancel。
- 打开时靠近 HUD，并保持在当前屏幕工作区内。

设置 UI 规则：

- 删除单独的 `Change` 按钮。
- 颜色预览块本身可点击。
- 点击颜色预览块打开系统 `ColorDialog`。
- Hex 输入框仍支持手动编辑。
- Hex 必须是 `#RRGGBB`。
- 非法 hex 保存时应提示或阻止保存。
- 设置窗口宽度应随删除 `Change` 按钮后相应缩小。

推荐设置项：

```text
Auto Refresh
Enable Antigravity
Colors
7d Color
5h Color
Track Color
Track Border
Reset
Cancel
Save
```

### 6.7 `SettingsStore.cs`

职责：

- 读取设置。
- 保存设置。
- 缺失或非法时回退默认值。
- 创建配置目录。

设置路径：

```text
%APPDATA%\CodexQuotaHud\settings.json
```

不要将用户设置保存在仓库或 exe 同级目录。

### 6.8 `AppSettings.cs`

典型字段：


## 7. Codex Provider

Codex 读取方式：

```text
每次刷新
  -> 启动 codex app-server --listen stdio://
  -> initialize
  -> account/rateLimits/read
  -> 解析额度
  -> 结束子进程
```

职责：

- 查找 Codex CLI。
- 启动 app-server。
- 通过 stdin/stdout 交互。
- 读取 JSON。
- 解析 `7d` 和 `5h`。
- 返回统一 provider snapshot。

Codex CLI 路径优先级：

1. `CODEX_CLI_PATH`。
2. `%LOCALAPPDATA%\OpenAI\Codex\bin\codex.exe`。
3. `%LOCALAPPDATA%\OpenAI\Codex\bin\` 下的版本化目录。
4. PATH 中的 `codex`。

失败处理：

- HUD 显示简短 `Failed`。
- 详细错误写入 debug log。
- 不要把大段错误堆进 HUD。

## 8. AGY / Antigravity Provider

### 8.1 产品策略

AGY 默认关闭。用户在设置中启用后，使用 **Managed AGY** 模式。

```text
Enable Antigravity = false 默认值
启用后：HUD 启动并管理隐藏 agy.exe
```

### 8.2 Managed AGY 生命周期

基本流程：

```text
HUD 启动或设置启用 AGY
  -> 检查是否需要启动托管 agy.exe
  -> 创建专用工作目录
  -> 隐藏启动 agy.exe
  -> 等待本地 quota endpoint ready
  -> 记录 PID / StartTime / Port / WorkingDirectory
  -> 周期性读取 RetrieveUserQuotaSummary
  -> HUD 退出时只关闭自己启动的 agy.exe
```

建议托管工作目录：

```text
%LOCALAPPDATA%\CodexQuotaHud\agy-provider
```

### 8.3 为什么使用常驻托管，而不是 one-shot

AGY CLI 是 TUI/CLI，不是 Codex 那样明确的 stdio app-server。首次或特定环境下可能出现：

- 登录交互。
- trust folder 交互。
- 权限提示。
- TUI 初始化。

因此 AGY 不建议每次刷新都启动 / 查询 / 杀掉。常驻托管更稳定，也更容易复用本地 quota endpoint。

### 8.4 隐藏启动策略

日常运行可尝试隐藏启动：

```text
UseShellExecute = false
CreateNoWindow = true
WindowStyle = Hidden
```

但隐藏启动不保证首次配置成功。若 endpoint 长时间不可用，应提示用户手动运行一次：

```powershell
agy
```

完成登录 / trust 后，再让 HUD 托管隐藏启动。

### 8.5 AGY endpoint

AGY Provider 请求：

```text
POST https://127.0.0.1:{port}/exa.language_server_pb.LanguageServerService/RetrieveUserQuotaSummary
```

Headers：

```text
Content-Type: application/json
Connect-Protocol-Version: 1
```

Body：

```json
{
  "metadata": {
    "ideName": "antigravity",
    "extensionName": "antigravity",
    "locale": "en",
    "ideVersion": "unknown"
  }
}
```

AGY CLI 实测不需要 `X-Codeium-Csrf-Token`。

### 8.6 AGY 解析规则

返回结构中关注：

```text
response.groups[]
  Gemini Models
    gemini-weekly
    gemini-5h
```

映射规则：

```text
gemini-weekly.remainingFraction -> AGY 7d RemainingPercent
gemini-weekly.resetTime         -> AGY 7d ResetAt

gemini-5h.remainingFraction     -> AGY 5h RemainingPercent
gemini-5h.resetTime             -> AGY 5h ResetAt
```

解析器必须宽松：

- 优先依赖 `bucketId` 和 `remainingFraction`。
- 不要强依赖 `description`、`window`、`displayName`。
- 兼容 `remainingFraction` 直接字段。
- 兼容 `remaining.remainingFraction` 嵌套字段。
- resetTime 缺失时不应崩溃。

### 8.7 AGY 错误处理

常见错误：

| 场景 | UI 处理 |
|---|---|
| 未启用 Antigravity | 不显示 AGY 模块 |
| 找不到 agy.exe | AGY card 显示 Not installed / HUD toast 提示安装 |
| agy.exe 启动但 endpoint 不 ready | 提示完成登录 / trust |
| 请求失败 | 重新 discovery，必要时重启托管进程 |
| 字段缺失 | 宽松解析并显示 Failed |
| AGY 进程退出 | 根据设置尝试重启，或显示 Offline |

HUD 附近提示应为非模态 toast，不要使用打断式 MessageBox。

同一种错误不要每个刷新周期重复弹出。

### 8.8 进程安全

禁止：

```text
kill all agy.exe
```

只允许关闭 HUD 自己启动并记录的托管进程。

退出时应校验：

```text
PID == ManagedAgyPid
进程名仍为 agy.exe
StartTime 与记录匹配
WorkingDirectory / command line 与托管标记匹配
```

避免误杀用户正在使用的 AGY CLI。


- 不显示百分比。


## 10. 刷新行为与防闪烁

支持手动刷新和自动刷新。

自动刷新选项：

```text
30 sec
1 min
5 min
10 min
20 min
```

默认：

```text
1 min
```

刷新状态：

```text
Refreshing...
Updated
Failed
```

防闪烁原则：

- 不要每次刷新 `Controls.Clear()`。
- 不要每次刷新重建 provider card。
- 不要每次刷新调用会调整窗口大小的 layout。
- 不要无差别 `Invalidate()` 整个 Form。
- 只更新发生变化的 label / progress bar。
- `QuotaBarControl.Percent` 未变化时不要重绘。
- provider 数量变化时才重新 layout。
- 折叠 / 展开状态变化时才调整窗口尺寸。
- 控件开启双缓冲。

目标：

```text
刷新额度时只变数字、时间、进度条，不让整个 HUD 闪一下。
```

## 11. 设置行为

设置窗口应打开在 HUD 附近。

定位规则：

1. 优先放在 HUD 右下侧。
2. 右侧空间不够则放左侧。
3. 下方空间不够则放上方。
4. 始终保持在当前屏幕工作区内。

默认设置：

```text
Auto Refresh:       1 min
Enable Antigravity: false
7d Color:           #4EA1FF
5h Color:           #C2410C
Track Color:        #303740
Track Border Color: #7A8796
```

颜色行为：

- 点击颜色预览块打开调色盘。
- 删除 `Change` 按钮。
- 缩小设置窗口宽度。
- Hex 输入框与颜色预览块保持同步。
- Reset Defaults 更新表单字段。
- Save 持久化设置并应用到 HUD。
- Cancel 不保存。

## 12. 日志与安全

不要在 UI 或日志中完整打印：

- CSRF token。
- 完整 command line。
- 完整 raw JSON。
- 账号邮箱。
- 未来可能出现的用户身份信息。

允许记录脱敏信息：

```text
endpoint=pid=1234, port=5678, source=managed-agy
csrf=8cdea676...cb4b
```

TLS 注意：

- AGY / App Local 的本地接口可能使用自签 HTTPS。
- 证书绕过只能限制在 `127.0.0.1`。
- 不要全局关闭 TLS 校验。

进程注意：

- 只关闭 HUD 自己启动的 AGY。
- 不关闭用户已有的 AGY。
- 强制退出 HUD 可能不会清理托管 AGY，调试时需要手动检查。

- 手动输入 hex 能更新颜色块。
- 非法 hex 被拦截或提示。
- Reset Defaults 恢复默认值。
- 5h 默认颜色为 `#C2410C`。

### 防闪烁测试

- 自动刷新时 HUD 不整体闪烁。
- 手动刷新时 HUD 不整体闪烁。
- 只有变化的进度条和文字更新。
- provider 数量不变时不重建整个 UI。

### Release 测试

- push 到 `main` 产生两个 Actions artifacts。
- tag push 创建或更新 GitHub Release。
- Release 包含两个 exe。
- no-.NET 版本体积较小。
- with-.NET 版本体积较大。

## 17. 安全变更策略

### 结构调整前

1. 确认当前 build 通过。
2. 只移动文件，不改逻辑。
3. 再次 build。
4. 结构调整和行为变更分开提交。

### Provider 修改前

1. 不同时修改 UI。
2. 先增加必要诊断。
3. build。
4. 使用真实本地 CLI 测试。

### UI 修改前

1. 不修改 provider。
2. 保持 public model 属性兼容。
3. build。
4. 手动测试展开态和折叠态。

### AGY 生命周期修改前

1. 明确区分用户已有 AGY 与 HUD 托管 AGY。
2. 只关闭 HUD 自己启动的 PID。
3. 不使用 kill all。
4. 测试 HUD 正常退出和强制退出后的状态。
