# Codex Quota HUD 项目开发说明

本文面向未来维护者和 AI 编程代理，用于说明当前项目结构、关键行为、生命周期管理、诊断日志、测试要求和安全边界。

项目已经从单一 Codex HUD 扩展为：

```text
Codex provider + 可选 Managed AGY provider + session-based diagnostics
```

后续修改必须保持模块化，避免把所有逻辑重新塞回 `MainHudForm.cs`。

---

## 1. 项目概述

Codex Quota HUD 是一个 Windows 原生桌面 HUD 工具，使用 **C# + WinForms** 开发。

当前核心能力：

```text
读取本机额度 -> 统一成 provider/bucket 模型 -> 以 HUD 展示 -> 可选写入诊断日志
```

当前 provider：

- `Codex`：通过本机 Codex CLI `app-server --listen stdio://` 读取 `7d` / `5h` 额度。
- `AGY · Gemini`：可选启用，通过 HUD 托管启动的 Antigravity CLI `agy.exe` 读取 Gemini `7d` / `5h` 额度。

Codex provider 当前默认使用 plugins-disabled launch mode：

```text
codex -c features.plugins=false -c features.remote_plugin=false app-server --listen stdio://
```

该模式用于避免 Codex app-server 启动时进行 curated plugins sync，从而降低 `git.exe` / `git-remote-https.exe` / Git Credential Manager 弹窗概率。

产品仍然是小型 HUD，不是完整账号仪表盘。

## 3. 技术栈

```text
语言：C#
UI：WinForms
目标框架：net8.0-windows
运行目标：Windows x64
构建：dotnet build
发布：dotnet publish
CI：GitHub Actions / windows-latest
```

`System.Management` 当前用于 Windows WMI 进程诊断，例如读取 PID、PPID、CommandLine 和 ExecutablePath。

WinForms 适合当前项目，因为 HUD、托盘、置顶窗口、无边框窗口、拖动、吸附和自绘进度条都可以用原生能力完成。

## 5. 核心架构

### 5.1 Provider 模型

不要让 UI 直接依赖 Codex 或 AGY 的原始协议字段。数据流应该是：

```text
IQuotaProvider
  -> ProviderQuotaSnapshot
    -> QuotaBucketSnapshot[]
      -> UI 控件展示
```

`ProviderQuotaSnapshot` 应包含：

- `ProviderId`
- `DisplayName`
- `Subtitle`
- `Source`
- `Status`
- `Buckets`
- `UpdatedAt`
- `CheckedAt`
- `ChangedAt`
- `ErrorMessage`
- `IsManagedProcess`
- `ProcessId`
- `Port`

`QuotaBucketSnapshot` 应包含：

- `Id`
- `Label`
- `ShortLabel`
- `RemainingPercent`
- `ResetAt`
- `RawResetTimeUtc`

UI 层统一显示 `7d` / `5h`，即使 AGY 内部实际字段是 `gemini-weekly` / `gemini-5h`。

## 6. 主要模块说明

### 6.1 `Program.cs`

职责：

- 初始化 WinForms。
- 创建并运行 `MainHudForm`。

不要在此处加入额度读取、provider 生命周期、UI 细节或发布逻辑。

---

### 6.2 `MainHudForm.cs`

主 HUD 窗口。

职责：

- 管理展开态和吸附/折叠态。
- 管理拖动、靠顶吸附、悬停展开。
- 管理托盘图标和托盘菜单。
- 调用 `QuotaPoller` 刷新 provider snapshot。
- 复用 `ProviderCardControl` 和 `CollapsedProviderRowControl`。
- 显示 AGY 错误 toast。
- 管理退出流程。
- 在 Settings 保存后应用 `EnableDiagnosticLogging` 等设置。

重要规则：

- 不要每次刷新都 `Controls.Clear()` 再重建所有控件。
- Provider card / collapsed row 应尽量复用。
- 只有 provider 数量变化、展开/折叠状态变化、footer 高度变化时才重新 layout。
- 普通额度刷新只更新文字、时间、进度条 percent。
- 退出时必须先停止 timer、取消刷新、释放 provider，再关闭托盘图标和窗体。

---

### 6.3 `QuotaPoller.cs`

统一调度 provider 刷新。

职责：

- 永远刷新 Codex。
- 只有设置启用时才刷新 AGY。
- 使用 `_refreshGate` 避免并发刷新。
- 支持退出时 cancellation。
- Dispose 时释放 provider。

当前刷新顺序：

```text
Codex -> AGY if enabled
```

注意：

- 不要重新加入固定的 AGY cold-start delay。
- 如果需要减少启动干扰，应通过更好的 provider 调度或日志诊断解决，而不是强行让窗口慢 2 秒以上。

建议：

- `RefreshAsync` 接受 `CancellationToken`。
- 退出时先 cancel，再 dispose。
- `DisposeAsync` 内部 await 应使用 `ConfigureAwait(false)`，避免 WinForms UI 线程死锁。

---

### 6.4 `CodexQuotaProvider.cs`

Codex provider 适配层。

职责：

- 调用 `CodexQuotaReader.ReadAsync()`。
- 将 Codex 专用 `QuotaSnapshot` 映射为统一 `ProviderQuotaSnapshot`。
- 将 `SevenDay` / `FiveHour` 映射为 `7d` / `5h` bucket。
- 维护 `ChangedAt`。

---

### 6.5 `CodexQuotaReader.cs`

读取 Codex CLI 额度。

当前模式是 one-shot 子进程：

```text
每次刷新
  -> 启动 codex app-server --listen stdio://
  -> 发送 initialize
  -> 发送 account/rateLimits/read
  -> 读取 JSON response
  -> 解析 result
  -> kill 子进程
```

路径解析顺序：

1. `CODEX_CLI_PATH`
2. `%LOCALAPPDATA%\OpenAI\Codex\bin\codex.exe`
3. `%LOCALAPPDATA%\OpenAI\Codex\bin\` 下的版本化子目录
4. PATH 中的 `codex`

#### Codex launch mode

当前默认：

```text
DisableCodexPluginsForQuotaRead = true
```

优先启动：

```text
codex -c features.plugins=false -c features.remote_plugin=false app-server --listen stdio://
```

如果失败，自动 fallback：

```text
codex app-server --listen stdio://
```

日志应记录：

```text
[CODEX-DIAG] codex launch mode=plugins-disabled
[CODEX-DIAG] codex launch mode=plugins-disabled succeeded
[CODEX-DIAG] codex child process summary ... usedFallback=False rateLimitReadSucceeded=True
```

如果 fallback：

```text
[CODEX-DIAG] codex launch mode=plugins-disabled failed; falling back to legacy launch ...
[CODEX-DIAG] codex child process summary ... usedFallback=True ...
```

#### Git 非交互环境变量

Codex 子进程应设置：

```text
GIT_TERMINAL_PROMPT=0
GCM_INTERACTIVE=false
GCM_GUI_PROMPT=false
GIT_ASKPASS=
SSH_ASKPASS=
```

这些变量只影响 HUD-launched Codex 子进程，不修改系统环境变量。

#### Process diagnostics

Codex reader 负责记录：

```text
before-codex-start
after-codex-start
codex-refresh monitor
monitor-cancel-requested
before-codex-kill
after-codex-kill
child process summary
```

当前策略：

- Codex refresh 结束后立刻 cancel monitor。
- 不要让 Codex monitor 继续跑 30 秒并把 AGY 启动记录成 Codex unrelated new process。
- 如果诊断日志关闭，不应写大量 process snapshots。

---

### 6.6 `ManagedAgyQuotaProvider.cs`

AGY provider 适配层。

职责：

- 检查 AGY 是否启用。
- 调用 `ManagedAgyProcess.EnsureRunningAsync()` 确保 HUD 托管的 AGY 正在运行。
- 发现或复用本地 quota endpoint 端口。
- 调用 `RetrieveUserQuotaSummary`。
- 解析 `gemini-weekly` / `gemini-5h`。
- 映射为 UI 的 `7d` / `5h`。
- 维护 `CheckedAt` / `ChangedAt`。
- 在 endpoint 不可用时返回 Offline / Failed snapshot。

推荐优化：

- 复用 provider 级别 `HttpClient`，避免每次刷新都创建。
- endpoint 不可用时设置 backoff，避免每轮刷新都完整 discovery 15 秒。
- 连续 not ready 后可关闭托管 AGY，并提示用户手动运行一次 `agy`。

---

### 6.7 `ManagedAgyProcess.cs`

管理 HUD 自己启动的 `agy.exe`。

职责：

- 解析 `agy.exe` 路径。
- 使用专用工作目录启动 AGY。
- 支持隐藏启动。
- drain stdout/stderr，避免管道阻塞。
- 记录托管进程的 PID、启动时间、路径、工作目录。
- HUD 退出时只关闭自己启动的 AGY。
- AGY 意外退出后，下次刷新可以重新启动。

重要规则：

```text
禁止 kill all agy.exe
只允许关闭 HUD 自己启动且身份匹配的 agy.exe
```

判断托管进程时建议检查：

- PID 是否是记录的 PID。
- 进程名是否为 `agy`。
- StartTime 是否匹配。
- MainModule 路径是否匹配。
- 工作目录或启动标记是否匹配。

当前保留：

```text
CloseManagedAgyOnExit = true
StartAgyHidden = true
AntigravityMode = managed-agy
```

如未来不再开放这些设置，应删除未使用字段，避免误导维护者。

---

### 6.8 `AgyEndpointDiscovery.cs`

发现 AGY 本地监听端口。

职责：

- 根据 PID 查找 TCP Listen 端口。
- 对候选端口进行 quota endpoint probe。

当前可用方式是调用：

```text
netstat -ano -p tcp
```

并解析 PID 对应端口。

注意：

- 不要假设固定端口。
- 如果 endpoint 连续不可用，应 backoff，避免频繁启动 `netstat`。
- discovery 必须支持 cancellation。
- `netstat` 相关日志要受 diagnostic logging 开关控制。

---

### 6.9 `AntigravityQuotaParser.cs`

解析 AGY quota JSON。

核心字段：

```text
response.groups[].buckets[]
bucketId == gemini-weekly
bucketId == gemini-5h
remainingFraction
resetTime
```

映射规则：

```text
gemini-weekly -> UI 7d
gemini-5h     -> UI 5h
remainingPercent = remainingFraction * 100
resetTime UTC -> 本地时间显示
```

解析器必须宽松：

- 不要强依赖 `description`。
- 不要强依赖 `displayName`。
- 不要强依赖 `window`。
- 兼容 `remainingFraction` 在 bucket 顶层或 `remaining.remainingFraction` 下。

---

### 6.10 `DebugLogger.cs`

统一日志服务。

当前策略：

```text
日志目录：%LOCALAPPDATA%\CodexQuotaHud\logs\
每次启动创建：debug-YYYY-MM-DD_HH-mm-ss.log
单个 session log 最大：2 MB
最多保留：最近 15 个 debug-*.log
超过 2 MB：写一次提示后停止继续写入
latest.log：不做
文件名：不写 PID
日志第一行：写 PID
```

重要行为：

- `EnableDiagnosticLogging = true` 默认开启。
- `Info` / `Error` 类型日志应强制写入。
- 详细 `Diagnostic` / `ProcessDiagnostics` 日志受设置开关控制。
- `ApplySettings` 后应记录当前 diagnostic logging 状态。
- 日志清理在启动时执行。
- 不要无限追加到同一个 `debug.log`。

安全要求：

- 不记录 token。
- 不记录 raw JSON。
- 不记录用户 prompt。
- 不记录完整环境变量。
- 本机路径和非敏感命令行片段可以用于诊断，但要截断和脱敏。

---

### 6.11 `ProcessDiagnostics.cs`

进程诊断模块。

职责：

- 读取有诊断价值的进程。
- 通过 WMI 获取 PID、PPID、进程名、路径、命令行。
- 记录 provider 相关进程快照。
- 监控短时间内新出现的子进程。
- 识别 `codex` / `agy` / `git` / `git-remote-https` / `conhost` / `node` / `netstat` 等进程。

关系字段：

```text
target      当前目标进程
related     provider 相关但不是目标子进程
descendant  目标进程的子孙进程
unrelated   无关进程
```

注意：

- Codex monitor 应在 Codex refresh 结束后立即 cancel。
- 不要让 Codex monitor 把之后启动的 AGY 记录为 Codex new process。
- `conhost.exe` 仍可能出现；当前不以彻底消除它为目标。
- 如果 summary 中 `conhostDescendantSeen` 与 snapshot 不一致，优先以 snapshot 为准。该问题不应阻塞当前 Git 弹窗修复。

## 8. 刷新行为

自动刷新选项：

```text
30 sec
1 min
5 min
10 min
20 min
```

默认：

```text
1 min
```

刷新规则：

- 手动刷新通过右键菜单 `Refresh Now`。
- 自动刷新由 WinForms Timer 触发。
- 不允许并发刷新。
- 刷新时只更新变化的控件。
- 退出时必须取消正在进行的刷新。
- AGY 启动不应再人为固定延迟 2 秒。

防闪烁要求：

- Provider card / row 只创建一次并复用。
- `QuotaBarControl.Percent` 相同则不重绘。
- Label 文本相同则不重新赋值。
- 不要每次刷新都改变 Form Size。
- 只有 layout 真实变化时才 `ShowCurrentView(force: true)`。

## 9. 退出与生命周期要求

这是当前项目最重要的稳定性规则。

### 9.1 不要在 Dispose 中同步等待异步释放

禁止在 UI 线程 `Dispose()` 中使用：

```csharp
_poller.DisposeAsync().AsTask().GetAwaiter().GetResult();
```

这可能导致：

- 托盘图标已经消失。
- HUD 残留在桌面。
- 窗口无法移动/选中。
- 进程未退出。
- exe 被锁定，导致下一次 `dotnet run` 无法覆盖。

### 9.2 推荐退出流程

```text
ExitApplicationAsync
  -> 设置 _isExiting = true
  -> 禁用托盘菜单，避免重复点击
  -> 停止 refresh timer
  -> Cancel 应用级 CancellationTokenSource
  -> await 当前 refresh / provider dispose
  -> 关闭 HUD 自己启动的 AGY
  -> 隐藏并释放 NotifyIcon
  -> DebugLogger.Shutdown()
  -> Close / Application.ExitThread
```

所有后台 async dispose 链路建议使用：

```csharp
await SomethingAsync().ConfigureAwait(false);
```

避免捕获 WinForms UI 同步上下文。

### 9.3 AGY 退出要求

- HUD 退出时只关闭自己启动的 AGY。
- 不允许 kill 用户手动打开的 AGY。
- 如果 AGY 意外退出，下一轮刷新应重新启动。
- 如果 AGY endpoint 长期不可用，应 backoff，并提示用户手动运行一次 `agy`。

### 11.2 日志保留策略

```text
每次启动创建一个 session log
文件名：debug-YYYY-MM-DD_HH-mm-ss.log
单个文件上限：2 MB
最多保留：最近 15 个 debug-*.log
超过 2 MB：写 suppress 提示后停止写入
latest.log：不做
```

### 11.3 日志开关

Settings 中有：

```text
Enable diagnostic logging
```

默认开启。

开关含义：

```text
On:
  写 Codex / AGY process diagnostics、endpoint discovery、netstat、launch mode 等详细日志。

Off:
  不写大量 process diagnostics。
  仍可保留必要的 APP / ERROR / session lifecycle 信息。
```

### 11.4 安全边界

不要记录：

- 完整 CSRF token。
- Authorization header。
- API key。
- 完整 raw JSON。
- 用户 prompt。
- 账号邮箱。
- 完整环境变量。

允许记录但要脱敏或截断：

- 本机路径。
- 进程名。
- PID / PPID。
- 非敏感命令行片段。
- Codex launch mode。

AGY CLI provider 当前不需要 CSRF header；未来如果新增 App Local provider，要特别注意 token 脱敏。

TLS 证书绕过只允许用于：

```text
https://127.0.0.1:<port>
```

不要全局关闭 TLS 校验。

### 13.4 Diagnostics 测试

- 每次启动生成一个新的 `debug-*.log`。
- 文件名不包含 PID。
- 第一行包含 `session started pid=...`。

- 单个日志超过 2 MB 后停止写入。
- 日志目录最多保留最近 15 个 `debug-*.log`。
- Settings 关闭 `Enable diagnostic logging` 后，大量 `[PROCESS-DIAG]` 不再写入。
- Codex refresh 结束后出现 `monitor-cancel-requested` / `monitor-cancelled`。

### 13.5 Exit 测试

这是必须测试项。

- 右键托盘 `Exit` 后：
  - 托盘图标消失。
  - HUD 窗口消失。
  - `CodexQuotaHud.exe` 进程退出。
  - HUD 托管的 `agy.exe` 退出。
  - 不留下无法移动/无法选中的残影窗口。
- 退出后立即执行 `dotnet run` 不应出现 exe 被锁定。
- 日志末尾应出现：

```text
[APP] exit requested
[APP] poller dispose started
[AGY-DIAG] stopping managed agy ...
[AGY-DIAG] stopped managed agy ...
[APP] poller dispose completed
[APP] session ended ...
```

如果出现：

```text
CodexQuotaHud.exe is being used by another process
```

说明旧进程未退出，应检查退出流程。

## 16. 安全修改策略

结构性调整：

1. 先确认当前 build 通过。
2. 只移动文件，不改逻辑。
3. 再 build。
4. 行为修改另起提交。

Quota reader 修改：

1. 不改 UI。
2. 加必要诊断。
3. build。
4. 用真实本地 CLI 测试。
5. 检查日志中 `usedFallback`、`gitDescendantSeen`、`rateLimitReadSucceeded`。

UI 修改：

1. 不改 provider 协议。
2. 保持 public model 稳定。
3. build。
4. 手动测试展开态、折叠态、吸附态。

Exit / 生命周期修改：

1. 不改额度解析。
2. 加 cancellation。
3. 避免 UI 线程同步等待 async。
4. build。
5. 测试 Exit 后无残留进程、无 exe 锁定。

Diagnostics 修改：

1. 不记录敏感信息。
2. 保持 2 MB 单文件上限。
3. 保持 15 个 session log 保留策略。
4. Settings 关闭 diagnostic logging 后不要写大量 process snapshot。
5. build。
6. 上传日志前人工检查敏感信息。
